## ⚠️ قوانین مهم

### 1. پاک کردن داده‌های قبلی (خیلی مهم!):
**⚠️ هرگز داده‌های برندهای قبلی را نگه ندار!**

- **قبل از شروع:** تمام داده‌های قبلی را از فایل پایه پاک کن:
  - جداول را خالی کن یا با داده‌های جدید جایگزین کن
  - نمودارها را با داده‌های جدید به‌روزرسانی کن
  - کارت‌های آماری را با داده‌های جدید به‌روزرسانی کن
  - نام برند را در همه جا به‌روزرسانی کن
  - لوگو را به‌روزرسانی کن

- **بررسی نهایی:** مطمئن شو که هیچ داده‌ای از برندهای قبلی باقی نمانده:
  - در جداول
  - در نمودارها
  - در کارت‌های آماری
  - در متن‌ها و توضیحات

### 2. پردازش انعطاف‌پذیر جداول:
- **جداول می‌توانند هر ساختاری داشته باشند** - شما باید ساختار را از header استخراج کنید
- **نام ستون‌ها را تغییر نده** - همانطور که کاربر داده نمایش بده
- **اگر ستون جدیدی در جدول وجود دارد** → آن را هم اضافه کن
- **اگر ستونی وجود ندارد** → محاسبه نکن یا از ستون مناسب دیگر استفاده کن

### 3. اضافه کردن صفحات/داده‌های جدید:
- **اگر کاربر جدول جدیدی داد:**
  - صفحه جدید با عنوان مناسب اضافه کن
  - جدول را با همان ساختاری که کاربر داده بساز
  - دکمه "بیشتر" را اگر نیاز بود اضافه کن

- **اگر کاربر نمودار جدیدی خواست:**
  - نمودار را با داده‌های ورودی بساز
  - نوع نمودار را از داده‌ها یا درخواست کاربر تشخیص بده

- **اگر کاربر صفحه جدیدی خواست:**
  - صفحه را با محتوای مناسب اضافه کن
  - ساختار را از داده‌های ورودی یا درخواست کاربر بگیر

### 4. دکمه "بیشتر":
- اگر تعداد ردیف‌ها کمتر از 10 → همه را نمایش بده (بدون دکمه بیشتر)
- اگر 10 ردیف یا بیشتر → 10 ردیف اول نمایش بده + بقیه با `hidden-rows` + دکمه "بیشتر"

### 5. صفحات پیش‌فرض:
- **صفحه گزارش کلیدواژه‌ها را همیشه حذف کن** (مگر اینکه کاربر صراحتاً بخواهد)
- **صفحات بیلبورد جزئی:**
  - اگر گزارش کلی داده شده → صفحات بیلبورد جزئی را حذف کن
  - اگر گزارش جزئی داده شده → صفحات را نگه دار

### 6. ساختار داینامیک:
- **اگر کاربر داده جدیدی داد که در ساختار پیش‌فرض نیست:**
  - صفحه/بخش جدید اضافه کن
  - ساختار را از داده‌های ورودی تشخیص بده
  - نام و عنوان را از کاربر بگیر یا بر اساس داده‌ها تولید کن

## 🔄 مرحله 4: پردازش انعطاف‌پذیر جداول

### 4.0 قوانین کلی پردازش جدول:

**برای هر جدولی که کاربر می‌فرستد:**

1. **استخراج ساختار:**
   - خط اول جدول = header (نام ستون‌ها)
   - خطوط بعدی = داده‌ها
   - ستون‌ها را با tab یا pipe (|) جدا کن

2. **تشخیص ستون‌ها:**
   - نام ستون‌ها را از header بگیر
   - نام ستون‌ها را همانطور که کاربر فرستاده نگه دار (فارسی/انگلیسی/ترکیبی)
   - نام ستون‌ها را تغییر نده

3. **نمایش داده‌ها:**
   - داده‌ها را دقیقاً همانطور که کاربر فرستاده نمایش بده
   - فرمت اعداد را حفظ کن (کاما، نقطه، ...)
   - فرمت درصدها را حفظ کن
   - ترتیب ستون‌ها را حفظ کن

4. **استفاده از ستون‌های خاص (اگر وجود دارد):**
   - اگر ستون "CTR" یا "Click-Through Rate" وجود دارد → برای کلاس ctr استفاده کن
   - اگر ستون "CPV" یا "Cost per View" وجود دارد → برای stat card استفاده کن
   - اگر ستون "CPC" یا "Cost per Click" وجود دارد → برای stat card استفاده کن
   - اگر ستون "نمایش‌ها" یا "Impressions" یا "Views" وجود دارد → برای نمودار استفاده کن

5. **اگر ستون خاصی وجود ندارد:**
   - محاسبه نکن یا از "-" استفاده کن
   - یا از ستون مناسب دیگر استفاده کن

## 🔄 مرحله 5: تولید محتوا

### 5.1 به‌روزرسانی اطلاعات برند:
- Title tag
- Cover page (لوگو، نام برند)
- تمام بخش‌هایی که نام برند دارند

### 5.2 به‌روزرسانی داده‌های خلاصه:
- **جدول خلاصه کلی کمپین:**
  - ساختار جدول را از داده‌های کاربر تشخیص دهید
  - ستون‌ها را از header جدول استخراج کنید
  - داده‌ها را دقیقاً همانطور که کاربر فرستاده نمایش دهید
  - اگر ستون‌های خاصی (مثل CPV، CPC) وجود دارد، از آنها استفاده کنید
  - اگر وجود ندارد، فقط ستون‌های موجود را نمایش دهید
  
- **نمودار سهمی:**
  - اگر ستون "نمایش‌ها" یا "Impressions" وجود دارد، از آن برای محاسبه درصد استفاده کنید
  - اگر وجود ندارد، از ستون مناسب دیگر استفاده کنید یا نمودار را اضافه نکنید
  - درصدها را محاسبه کنید: `(campaign_value / total_value) * 100`
  
- **6 stat card:**
  - متریک‌ها را از جدول خلاصه استخراج کنید
  - اگر ستون "نمایش‌ها" وجود دارد → نمایش‌ها
  - اگر ستون "کلیک‌ها" وجود دارد → کلیک‌ها
  - اگر ستون "CTR" وجود دارد → CTR
  - اگر ستون "CPC" وجود دارد → CPC
  - اگر ستون "CPV" وجود دارد → CPV
  - اگر ستون "هزینه" یا "Cost" وجود دارد → خرجکرد کل
  - اگر ستونی وجود ندارد، stat card مربوطه را حذف کنید یا با "-" نمایش دهید

### 5.3 به‌روزرسانی نمودارها:
- **نمودار سهمی:**
  - ساختار جدول خلاصه را بررسی کنید
  - ستون مناسب برای محاسبه درصد را پیدا کنید (معمولاً "نمایش‌ها" یا "Impressions")
  - اگر وجود ندارد، از ستون مناسب دیگر استفاده کنید
  - درصدها را محاسبه کنید: `(campaign_value / total_value) * 100`
  
- **نمودار خطی:**
  - ساختار جدول ترند روزانه را بررسی کنید
  - ستون تاریخ را پیدا کنید (ممکن است نام‌های مختلف داشته باشد: تاریخ، Date، روز، ...)
  - ستون نمایش/Impressions را پیدا کنید
  - آرایه‌های `dates` و `impressions` را از داده‌های جدول استخراج کنید
  - فرمت تاریخ را حفظ کنید (ممکن است MM/DD/YYYY، YYYY-MM-DD، یا فرمت فارسی باشد)

### 5.4 ساخت جداول عملکرد:
برای هر جدول:
- **ساختار جدول را از داده‌های ورودی تشخیص دهید:**
  - ستون‌ها را از header جدول استخراج کنید
  - داده‌ها را دقیقاً همانطور که کاربر فرستاده نمایش دهید
  
- **کلاس‌های CTR (اگر ستون CTR وجود دارد):**
  - اگر ستون CTR وجود دارد، کلاس مناسب را اضافه کنید:
    - `ctr-low`: برای CTR کمتر از 1%
    - `ctr-high`: برای CTR بیشتر یا مساوی 1%
  - اگر ستون CTR وجود ندارد، کلاس اضافه نکنید
  
- **تعداد ردیف‌ها:**
  - اگر کمتر از 10 ردیف: همه را نمایش بده (بدون دکمه بیشتر)
  - اگر 10+ ردیف: 10 اول + بقیه با `hidden-rows` + دکمه "بیشتر"
  
- **فرمت داده‌ها:**
  - اعداد را همانطور که کاربر فرستاده نمایش دهید
  - فرمت درصدها را حفظ کنید
  - فرمت اعداد را حفظ کنید (کاما، نقطه، ...)

### 5.5 محاسبات خودکار:
محاسبات را بر اساس ستون‌های موجود در جدول انجام دهید:

- **درصد نمودار سهمی:**
  - اگر ستون "نمایش‌ها" یا "Impressions" وجود دارد: `(campaign_impressions / total_impressions) * 100`
  - اگر وجود ندارد، از ستون مناسب دیگر استفاده کنید
  
- **تعداد کاربران یکتا:**
  - اگر ستون "نمایش‌ها" یا "Impressions" در مجموع وجود دارد: `total_impressions * 0.38`
  - اگر وجود ندارد، محاسبه نکنید یا از ستون مناسب دیگر استفاده کنید
  
- **CPC متوسط:**
  - اگر ستون‌های "هزینه" و "کلیک‌ها" وجود دارد: `total_cost / total_clicks`
  - اگر وجود ندارد، محاسبه نکنید
  
- **CPV متوسط:**
  - اگر ستون‌های "هزینه" و "نمایش‌ها" وجود دارد: `total_cost / total_impressions`
  - اگر وجود ندارد، محاسبه نکنید

**⚠️ مهم:** همیشه ابتدا بررسی کنید که ستون مورد نیاز وجود دارد یا نه. اگر وجود ندارد، محاسبه نکنید یا از ستون مناسب دیگر استفاده کنید.
