## 📥 مرحله 1: دریافت و تحلیل ورودی

**⚠️ مهم:** کارت‌های خلاصه (آفر، CPV، مجموع ویو، مجموع پرداختی) را **خودتان از جدول استخراج کنید** - نیازی به ورودی جداگانه نیست!

**⚠️ مهم:** جدول می‌تواند هر ساختاری داشته باشد. شما باید:

### 🔍 نحوه پردازش جدول:

1. **استخراج header:**
   - خط اول جدول = نام ستون‌ها
   - ستون‌ها را با tab یا pipe (|) جدا کن
   - نام ستون‌ها را همانطور که کاربر فرستاده نگه دار

2. **استخراج داده‌ها:**
   - هر خط بعدی = یک ردیف داده
   - داده‌ها را با همان جداکننده (tab یا pipe) جدا کن
   - فرمت داده‌ها را حفظ کن

3. **ساخت HTML جدول:**
   - `<thead>` با ستون‌های header
   - `<tbody>` با ردیف‌های داده
   - ترتیب ستون‌ها را حفظ کن
   - فرمت اعداد را حفظ کن

4. **استفاده از ستون‌های خاص:**
   - اگر ستون "CPV" یا "Cost per View" وجود دارد → برای کارت خلاصه استفاده کن
   - اگر ستون "مجموع ویو" یا "Total Views" وجود دارد → برای کارت خلاصه استفاده کن
   - اگر ستون "مجموع پرداختی" یا "Total Payment" وجود دارد → برای کارت خلاصه استفاده کن
   - اگر وجود ندارد، محاسبه کن یا از ستون مناسب دیگر استفاده کن

**⚠️ نکته مهم:** 
- صفحه استراتژی **همیشه** از جدول پکیج استخراج می‌شود
- **کلیدواژه‌ها و موضوعات:** طبق قوانین در `MEDIAPLAN_INSTRUCTIONS.md`

#### 2.3 صفحه استراتژی هدفمندی:
- **همیشه صفحه استراتژی را اضافه کن** - استراتژی را از جدول پکیج استخراج کن
- **از جدول پکیج استخراج کن:**
  1. ستون "نوع محصول" را پیدا کن (بیلبورد دیجیتال استاندارد/پریمیوم)
  2. ستون "نوع هدفمندی" را پیدا کن (کیوردی/موضوعی/ریتارگتینگ)
  3. ترکیب‌های منحصر به فرد را پیدا کن
  4. برای هر ترکیب یک بخش در صفحه استراتژی بساز:

     - **بیلبورد دیجیتال استاندارد + کیوردی** → بخش استاندارد با کلیدواژه‌ها

## 🔄 مرحله 4: پردازش انعطاف‌پذیر جداول

### 4.0 قوانین کلی پردازش جدول:

**برای هر جدولی که کاربر می‌فرستد:**

1. **استخراج ساختار:**
   - خط اول جدول = header (نام ستون‌ها)
   - خطوط بعدی = داده‌ها
   - ستون‌ها را با tab یا pipe (|) جدا کن

2. **تشخیص ستون‌ها:**
   - نام ستون‌ها را از header بگیر
   - نام ستون‌ها را همانطور که کاربر فرستاده نگه دار (فارسی/انگلیسی/ترکیبی)
   - نام ستون‌ها را تغییر نده

3. **نمایش داده‌ها:**
   - داده‌ها را دقیقاً همانطور که کاربر فرستاده نمایش بده
   - فرمت اعداد را حفظ کن (کاما، نقطه، ...)
   - فرمت درصدها را حفظ کن
   - ترتیب ستون‌ها را حفظ کن

4. **استخراج کارت‌های خلاصه از جدول:**
   - **آفر:** از ستون "آفر" یا "Discount" یا محاسبه از "درصد تخصیص"
   - **CPV:** از ستون "CPV" یا "قیمت واحد" یا محاسبه: `مجموع پرداختی / مجموع ویو`
   - **مجموع ویو:** از ستون "مجموع ویو" یا "Total Views" یا محاسبه: `نمایش روزانه × تعداد روز`
   - **مجموع پرداختی:** از ستون "مجموع پرداختی" یا "Total Payment" یا محاسبه: `قیمت واحد × مجموع ویو`

5. **اگر ستون خاصی وجود ندارد:**
   - محاسبه کن یا از ستون مناسب دیگر استفاده کن

   - **استاندارد + موضوعی:** اگر در جدول "بیلبورد دیجیتال استاندارد" + "موضوعی" وجود دارد
   - **ریتارگتینگ:** اگر در جدول "ریتارگتینگ" وجود دارد

    4. **مجموع پرداختی (هزینه کل):**

       - اگر وجود ندارد، از ستون "قیمت واحد" × "مجموع ویو" محاسبه کنید
  
- **جداول**: هر پکیج یک جدول جداگانه
  - **ساختار جدول را از داده‌های ورودی تشخیص دهید:**
    - ستون‌ها را از header جدول استخراج کنید

     - عنوان: "🎯 بیلبورد دیجیتال پریمیوم"

     - چون "موضوعی" وجود دارد → موضوعات تخصصی اضافه کن:
       - اگر کاربر داده → از آن استفاده کن
       - اگر نداده → پیشنهاد بده و منتظر تایید بمان
   
   - **بخش بیلبورد دیجیتال استاندارد:**
     - عنوان: "📊 بیلبورد دیجیتال استاندارد"
     - توضیح: "برای بیلبورد دیجیتال استاندارد از موضوعات و کلیدواژه‌های **مرتبط و کمی عمومی‌تر** استفاده می‌کنیم..."
     - چون "کیوردی" وجود دارد → کلیدواژه‌های عمومی اضافه کن:
       - اگر کاربر داده → از آن استفاده کن
       - اگر نداده → پیشنهاد بده و منتظر تایید بمان
     - چون "موضوعی" وجود دارد → موضوعات عمومی اضافه کن:
       - اگر کاربر داده → از آن استفاده کن
       - اگر نداده → پیشنهاد بده و منتظر تایید بمان

## 🎯 خلاصه: پردازش انعطاف‌پذیر

### ✅ همیشه انجام بده:
- هر جدولی که کاربر می‌فرستد را بپذیر
- ساختار جدول را از header استخراج کن
- ستون‌ها را همانطور که کاربر فرستاده نمایش بده
- فرمت داده‌ها را حفظ کن
- **کارت‌های خلاصه را از جدول استخراج کن** (آفر، CPV، مجموع ویو، مجموع پرداختی)
- **صفحه استراتژی را همیشه از جدول استخراج کن:**
  - ستون "نوع محصول" و "نوع هدفمندی" را پیدا کن
  - ترکیب‌های منحصر به فرد را استخراج کن:
    - بیلبورد دیجیتال پریمیوم + کیوردی → بخش پریمیوم با کلیدواژه‌ها
    - بیلبورد دیجیتال پریمیوم + موضوعی → بخش پریمیوم با موضوعات
    - بیلبورد دیجیتال استاندارد + کیوردی → بخش استاندارد با کلیدواژه‌ها
    - بیلبورد دیجیتال استاندارد + موضوعی → بخش استاندارد با موضوعات
    - ریتارگتینگ → بخش ریتارگتینگ
  - برای هر ترکیب بخش مناسب در صفحه استراتژی بساز
  - کلیدواژه‌ها/موضوعات را از ورودی کاربر بگیر
  - اگر کاربر نداده → پیشنهاد بده و منتظر تایید بمان
- اگر ستون خاصی وجود دارد، از آن استفاده کن
- اگر وجود ندارد، محاسبه کن یا از ستون مناسب دیگر استفاده کن

### ❌ هرگز انجام نده:
- نام ستون‌ها را تغییر نده
- ستون‌های جدید اضافه نکن (مگر محاسباتی)
- فرمت داده‌ها را تغییر نده
- فرض نکن که جدول ساختار خاصی دارد
- صفحه استراتژی را حذف نکن - همیشه از جدول استخراج می‌شود
