# mid-300x100 — 300 × 100 Mid-Content Banner

## Frame

```
Width: 300px   Height: 100px   Direction: RTL   Body: transparent
Viewport-triggered (IntersectionObserver).
```

```css
* { margin: 0; padding: 0; box-sizing: border-box; }
html, body { width: 300px; height: 100px; overflow: hidden; background: transparent; }
body { font-family: 'Vazirmatn','Segoe UI',Tahoma,sans-serif; position: relative; }
```

**Single-message rule:** ONE thing to communicate. If the brief has three messages, pick the strongest. A 300×100 banner that tries to say two things says nothing.

## Tags — image-pixel

```javascript
window.ALL_EVENT_TYPES = { IMPRESSION:'impression', CLICK:'click', LOOP:'loop', VIEWABLE:'viewable' };
window.fire_tag = function(t) {
  try { new Image().src = `https://tag.yektanet.com/event?type=${t}&t=${Date.now()}`; } catch(e) {}
};
fire_tag(ALL_EVENT_TYPES.IMPRESSION);
```

Fire `VIEWABLE` inside the IntersectionObserver callback when `isIntersecting && ratio > 0.5`.

---

## Viewport Trigger — mandatory

Animation must NOT start on load. It starts when the banner enters the viewport.

```javascript
const tl = gsap.timeline({ paused: true });
// ... build timeline ...

const io = new IntersectionObserver((entries) => {
  entries.forEach(e => {
    if (e.isIntersecting && e.intersectionRatio > 0.5) {
      fire_tag(ALL_EVENT_TYPES.VIEWABLE);
      tl.play();
      io.disconnect();
    }
  });
}, { threshold: 0.5 });
io.observe(document.body);
```

Without the viewport trigger, the animation finishes before the user even scrolls to it.

## Layout Patterns

The frame is wide (3:1 aspect). Use horizontal split.

**Pattern A — Visual-Right (default):**
```
[LEFT ~65%: logo + headline + subheadline + CTA]   [RIGHT ~35%: hero image]
 bg panel covers the left zone                        visual floats outside panel
 logo badge top-right of panel                        orbital/rings behind hero (optional)
```

**Pattern B — Big Number (offer-driven):**
```
[LEFT: LARGE % or number as hero]   [RIGHT: logo + copy + CTA]
Use when the offer IS the message (0% commission, 50% off, etc.)
```

**Pattern C — Logo-Centric (brand awareness):**
```
[CENTER: large logo]   [small tagline below]   [CTA bottom-right]
Minimal bg, lots of transparency. Use for brand recall, not offer.
```

Default to A. Use B when there's a strong number. Use C only for pure brand campaigns.

## Animation Budget

- **Style:** Single continuous motion — not 4 scenes. Elements enter, settle, CTA pulses, loop.
- **Intro:** all elements visible ≤500ms of viewport entry
- **CTA pulse:** starts at ~0.8s, runs indefinitely (via `tl.call`)
- **Runtime:** 20–30s linger time before loop (viewer reads, lingers, CTA pulses)
- **Loop:** `tl.onComplete` → fire LOOP → reload

**Typical timeline sketch:**
```
0.0s  bg panel fades in
0.1s  hero image pops in (back.out ease)
0.3s  logo badge fades in
0.5s  headline image slides up
0.7s  subheadline fades in
0.9s  CTA fades in
1.0s  CTA tapesh pulse begins (via tl.call)
28s   tl.onComplete → LOOP + reload
```

## Format-Specific Constraints

- BG panel must NOT cover the full 300×100 frame — it should be a bounded rectangle (~60–65% of area)
- Logo MUST be self-backed (white logos on transparent need a dark pill; brand logos on light bg need a white pill)
- Headline/subheadline are often transparent PNG image assets — they need the panel as a backdrop, OR they must be hidden in scenes where no backdrop is present
- No character poke-outs — the frame is only 100px tall; heroes must fit within it
- CSS initial state: all animated elements start at `opacity: 0` — never show before viewport trigger fires
