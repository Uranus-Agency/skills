## Tags — image-pixel

```javascript
window.ALL_EVENT_TYPES = { IMPRESSION:'impression', CLICK:'click', LOOP:'loop', VIEWABLE:'viewable' };
window.fire_tag = function(t) {
  try { new Image().src = `https://tag.yektanet.com/event?type=${t}&t=${Date.now()}`; } catch(e) {}
};
fire_tag(ALL_EVENT_TYPES.IMPRESSION);
```

Fire `VIEWABLE` when IntersectionObserver fires with `ratio > 0.5`.

## Viewport Trigger — mandatory

```javascript
const tl = gsap.timeline({ paused: true });
// ... build timeline ...

const io = new IntersectionObserver((entries) => {
  entries.forEach(e => {
    if (e.isIntersecting && e.intersectionRatio > 0.5) {
      fire_tag(ALL_EVENT_TYPES.VIEWABLE);
      tl.play();
      io.disconnect();
    }
  });
}, { threshold: 0.5 });
io.observe(document.body);
```

## Square-Breath Rule — vertical stack

This format is essentially square (300×250). Do NOT use 70/30 horizontal splits (they produce thin awkward columns). Use vertical zones:

```
┌──────────────────────────────┐  y=0
│  BRAND ZONE  (~60px = 24%)   │  ← logo + short tagline / headline
├──────────────────────────────┤  y=60
│                              │
│    HERO ZONE  (~130px = 52%) │  ← main visual, big number, KV image
│                              │
├──────────────────────────────┤  y=190
│  ACTION ZONE  (~60px = 24%)  │  ← subheadline + CTA button
└──────────────────────────────┘  y=250
```

The bg panel typically covers the full height but NOT full width:
`width: 260–270px; height: 230–240px; top: 5–10px; right: 0–5px; border-radius: 16px`
= ~67% of frame area → satisfies 70/30 rule with hero floating at left edge outside panel.

## Layout Patterns

**Pattern A — Vertical Stack (default, ~60% of cases):**
- Brand zone: logo badge top-right, small headline
- Hero zone: large product image / 3D icon / KV, centered
- Action zone: subheadline + CTA pill

**Pattern B — Big Number (offer-driven):**
```
[logo]                 ← brand zone
    ٪۵۰ or 0%          ← hero zone: huge number (90–100px font or large image asset)
  تخفیف ویژه
[CTA button]           ← action zone
```
Use when the offer IS the visual (0% commission, ٪۵۰ تخفیف, etc.)

**Pattern C — Multi-Product Carousel:**
Only when user specifies multiple products AND viewability is already >30%.
Cycle every 3s with dot indicators.

Default to A. Use B when there's a strong number hero. Never use C for first deliveries.

## Animation Budget

- **First-second rule:** all primary elements visible ≤800ms of viewport entry
  (user averages 1.5s of attention on this slot — hook them immediately)
- **CTA pulse:** starts at ~1.2s, runs indefinitely (via `tl.call`, killed in `onComplete`)
- **Runtime:** 16–24s — long enough for message to land, short enough not to bore
- **Loop:** `tl.onComplete` → kill infinite tweens → fire LOOP → `setTimeout(() => location.reload(), 100)`

**Typical timeline sketch (Pattern A):**
```
0.0s  bg panel fades in (opacity 0→1, scale 0.96→1)
0.15s hero image pops in (back.out ease, y: 15→0)
0.35s orbital/decorative rings fade in
0.5s  logo badge slides in from top
0.65s headline fades up
0.85s subheadline fades in
1.0s  CTA fades in
1.2s  CTA tapesh pulse begins (tl.call)
20s   tl.onComplete → loop
```

## Format-Specific Constraints

- BG panel must NOT be full-frame (300×250). It must be bounded, ~260–270px wide, leaving a 30–35px left edge transparent for the hero to "float outside."
- All elements start `opacity: 0` in CSS — never visible before viewport trigger
- Logo on transparent background: always use a self-backed pill (dark pill for white logos, light pill for dark logos)
- Headline/subheadline as transparent PNG: must be over the panel or hidden; never floating on transparent body
- Hero element (main KV) positioned at left edge, partially outside panel — this creates the 70/30 split vertically
- Viewability recovery: Pattern A + viewport trigger is mandatory on first delivery. At 9% baseline, the user has maybe 1.5s of attention. Everything important must be visible by 800ms.
