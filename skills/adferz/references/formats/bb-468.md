## Layout Patterns

The strip is too narrow for vertical stacking. Everything must be horizontal and scannable left-to-right (RTL: right-to-left).

**Pattern A — Logo | Headline | CTA (default):**
```
[RTL reading direction →]
[CTA 100×26px] ← ← ← [headline ~260px] ← ← ← [logo 28px high]
left:10             center                       right:10
```
- Logo: `right: 10px`, `height: 28px`
- Headline: `right: 60px`, `width: 280px`, single line, font 13–14px
- CTA pill: `left: 10px`, `height: 26px`, `width: 95–110px`

**Pattern B — Hero Accent + Headline + CTA:**
```
[hero icon 48×48px]  [headline + tagline 2-line]  [CTA]
Overflows top/bottom by 6px for visual interest
```
- Hero sits at `top: 50%; transform: translateY(-50%)`, `right: 8px`
- Headline `right: 66px`, max 2 lines at 12px

**Pattern C — 2-Scene Narrative:**
- Scene 1 (0–5s): brand logo + tagline, minimal bg (~30%)
- Scene 2 (5–12s): value proposition + CTA, bg rises to ~55%
- Loop: total 12–14s

Use C when there's a meaningful message shift (brand → offer).

---

## Animation Budget

- **Scenes:** 2–3 max (strip is too small for complex arcs)
- **Runtime:** 8–14s
- **Intro:** all elements visible ≤700ms — viewer scans and moves on fast
- **CTA pulse:** starts at ~1s, `scale: 1.04–1.06` yoyo, `sine.inOut`
- **Loop:** `tl.onComplete` → LOOP tag → reload

---

## Format-Specific Constraints

- Font size: text ≤14px (strip height = 60px, text above 14px crowds the panel)
- Logo height: ≤30px max
- CTA: `height: 24–28px` — never taller
- Avoid animating elements vertically more than ±8px (frame is only 60px tall)
- No hero character poke-outs — there's no overflow zone in this format; use `overflow: hidden`
- Horizontal slide-ins (from left or right) work well; vertical drops look wrong in a strip
